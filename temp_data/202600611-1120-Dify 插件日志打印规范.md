# Dify 插件日志打印规范（tool_logging 使用手册）

> 配套文档：[Dify 插件日志打印查看方式和打印流程](./202600611-1120-Dify%20插件日志打印查看方式和打印流程.md)  
> 源码位置：`test-dify/plugin-iot-device-plugin/tools/tool_logging.py`  
> 验证环境：K8s 自托管 Dify、plugin-daemon 0.6.x、dify_plugin SDK 0.9.x  
> 文档版本：v2.0 | 2026-06-11

---

## 一、文档目的

本文是团队开发 Dify 插件时的 **日志与 HTTP 调用规范**，核心是一个可复用模块 `tool_logging.py`。

阅读本文后，开发者应能：

1. 将 `tool_logging.py` 复制到新插件项目中直接使用
2. 按规范编写运行期 `_invoke` 与配置期 `_fetch_parameter_options`
3. 在 `kubectl logs` 中正确查看 `plugin instance log`
4. 避免 `Multiple subclasses of Tool` 等常见启动错误

---

## 二、核心结论

| 问题 | 结论 |
|------|------|
| 能否用 `print` 或 `print(stderr)` | **不能**，服务器安装模式下不可见 |
| 正确日志方式 | SDK 官方 `plugin_logger_handler`，由 `ToolLog` 封装 |
| 日志在哪里看 | `kubectl logs` plugin-daemon Pod，关键字 `plugin instance log` |
| 如何减少重复代码 | 所有 Tool 继承 `SpringToolMixin` + `Tool` |
| 基类能否直接继承 `Tool` | **不能**，会导致插件启动失败（见第五节） |

---

## 三、六条打印规范

| 编号 | 规范 | 配置期 `_fetch_parameter_options` | 运行期 `_invoke` |
|------|------|-----------------------------------|------------------|
| R1 | **入参必须打印** | `parameter` + 脱敏 `credentials` | `tool_parameters` 脱敏全量 |
| R2 | **异常必须打印** | `self.log.info` + `self.log.exception` | 同左 |
| R3 | **HTTP 返回码必须打印** | `log_response` 自动打印 `status_code` | 同左 |
| R4 | **错误响应体必须打印** | 4xx/5xx 时打印 body 前 200 字符 | 同左 |
| R5 | **成功响应体打印前 200 字符** | 同左 | 同左 |
| R6 | **异常回传前端** | 不适用，返回 `[]` | `yield self.create_text_message(err)` |

`SpringToolMixin.run_http()` 与 `fetch_get_json()` 已内置 R2～R5。  
`require_param()` / `require_spring_url()` + `error_text()` 已内置 R6。

## 四、项目集成步骤

### 4.1 目录结构

```
your-plugin/
├── tools/
│   ├── __init__.py          # 空文件，使 tools 成为 Python 包
│   ├── tool_logging.py      # 本文完整源码，直接复制
│   ├── your_tool.py
│   └── your_tool.yaml
├── provider/
│   └── your_provider.py
└── manifest.yaml
```

### 4.2 依赖

`tool_logging.py` 依赖：

- `dify_plugin>=0.9.0`（含 `plugin_logger_handler`）
- `requests`

### 4.3 新增工具三步走

**第一步**：创建 YAML，`identity.name` 记为 `tool_name`。

**第二步**：创建 Python 文件，继承写法固定为：

```python
from dify_plugin import Tool
from dify_plugin.entities.tool import ToolInvokeMessage
from .tool_logging import SpringToolMixin

class YourTool(SpringToolMixin, Tool):
    tool_name = "your_tool_name"   # 与 YAML identity.name 一致

    def _invoke(self, tool_parameters):
        ...
```

**第三步**：`manifest.yaml` 版本号递增，重新打包上传。

## 五、重要约束：为什么不能 `class Xxx(LoggedSpringTool)`

Dify SDK 加载每个工具 `.py` 时，扫描模块内所有 **`Tool` 的子类**，要求 **每个文件只能有一个**。

若 `tool_logging.py` 中写 `class LoggedSpringTool(Tool)`，工具文件再 `from .tool_logging import LoggedSpringTool`，加载器会在同一模块命名空间看到两个 `Tool` 子类：

1. `LoggedSpringTool`（被 import 进来）
2. `YourTool`

启动报错：

```
Exception: Multiple subclasses of Tool in .../tools/your_tool.py
```

**正确做法**：`SpringToolMixin` **不继承** `Tool`，工具类写为：

```python
class YourTool(SpringToolMixin, Tool):  # Mixin 在前，Tool 在后
    tool_name = "your_tool_name"
```

## 六、tool_logging.py 完整源码

将以下文件保存为 `tools/tool_logging.py`，可按项目修改 logger 前缀（默认 `iot_device_plugin`）和 `SENSITIVE_KEYS`。

```python
"""Dify 插件统一日志与 Spring HTTP 工具基类。"""

from __future__ import annotations

import json
import logging
from typing import Any

import requests
from dify_plugin.config.logger_format import plugin_logger_handler

LOG_BODY_MAX_LEN = 200

SENSITIVE_KEYS = frozenset({
    "api_token",
    "password",
    "secret",
    "access_key",
    "private_key",
})


def truncate_for_log(text: str, max_len: int = LOG_BODY_MAX_LEN) -> str:
    if not text:
        return ""
    if len(text) <= max_len:
        return text
    return f"{text[:max_len]}...(共{len(text)}字符)"


def safe_params_for_log(params: dict[str, Any]) -> dict[str, Any]:
    safe: dict[str, Any] = {}
    for key, value in params.items():
        if key in SENSITIVE_KEYS:
            safe[key] = "***"
        elif isinstance(value, str) and len(value) > LOG_BODY_MAX_LEN:
            safe[key] = truncate_for_log(value)
        else:
            safe[key] = value
    return safe


def safe_credentials_for_log(credentials: dict[str, Any]) -> dict[str, Any]:
    return safe_params_for_log(credentials)


def spring_url_from_credentials(credentials: dict[str, Any]) -> str:
    return credentials.get("spring_service_url", "").rstrip("/")


def request_headers_from_credentials(
    credentials: dict[str, Any],
    *,
    json_content: bool = True,
) -> dict[str, str]:
    headers: dict[str, str] = {"Accept": "application/json"}
    if json_content:
        headers["Content-Type"] = "application/json"
    api_token = credentials.get("api_token", "")
    if api_token:
        headers["Authorization"] = f"Bearer {api_token}"
    return headers


class ToolLog:
    """单工具日志门面，封装 plugin_logger_handler 与常用打印方法。"""

    def __init__(self, tool_name: str) -> None:
        self.tool_name = tool_name
        self.logger = logging.getLogger(f"iot_device_plugin.{tool_name}")
        self.logger.setLevel(logging.INFO)
        if not self.logger.handlers:
            self.logger.addHandler(plugin_logger_handler)

    def info(self, msg: str) -> None:
        self.logger.info(f"[{self.tool_name}] {msg}")

    def exception(self, msg: str) -> None:
        self.logger.exception(f"[{self.tool_name}] {msg}")

    def invoke_enter(self, tool_parameters: dict[str, Any]) -> None:
        self.info("===== _invoke ENTER =====")
        self.info(f"tool_parameters = {safe_params_for_log(tool_parameters)}")

    def invoke_exit_success(self) -> None:
        self.info("===== _invoke EXIT success =====")

    def fetch_enter(self, parameter: str) -> None:
        self.info("===== _fetch_parameter_options ENTER =====")
        self.info(f"parameter = {parameter!r}")

    def fetch_exit(self) -> None:
        self.info("===== _fetch_parameter_options EXIT =====")

    def log_request(
        self,
        method: str,
        url: str,
        *,
        params: dict[str, Any] | None = None,
        payload: Any = None,
    ) -> None:
        self.info(f"准备请求 method={method} url={url}")
        if params:
            self.info(f"query_params = {safe_params_for_log(params)}")
        if payload is not None:
            if isinstance(payload, dict):
                self.info(f"payload = {safe_params_for_log(payload)}")
            else:
                self.info(f"payload = {truncate_for_log(str(payload))}")

    def log_response(self, response: requests.Response, label: str = "响应") -> None:
        status = response.status_code
        body_text = response.text or ""
        self.info(f"{label} 状态码 = {status}")
        if status >= 400:
            self.info(f"{label} 错误响应体 = {truncate_for_log(body_text)}")
        else:
            self.info(f"{label} 响应体 = {truncate_for_log(body_text)}")

    def error_text(
        self,
        title: str,
        params: dict[str, Any],
        detail: str,
        hint: str = "",
    ) -> str:
        lines = [
            f"错误：{title}",
            "━━━━━━━━━━━━━━━━",
            "请求参数",
            json.dumps(safe_params_for_log(params), ensure_ascii=False, indent=2),
            "━━━━━━━━━━━━━━━━",
            f"异常信息：{detail}",
        ]
        if hint:
            lines.append(f"建议：{hint}")
        return "\n".join(lines)

    def http_error_detail(self, exc: requests.exceptions.HTTPError, max_len: int = 500) -> str:
        if exc.response is None:
            return str(exc)
        body = exc.response.text
        detail = truncate_for_log(body, max_len=max_len)
        try:
            detail = exc.response.json().get("error", detail)
        except Exception:
            pass
        return detail


class SpringToolMixin:
    """
    Spring HTTP 与日志能力 Mixin，不可直接继承 Tool。

    Dify 加载工具模块时要求每个文件只有一个 Tool 子类；
    若本类继承 Tool，import 后会导致 Multiple subclasses of Tool 启动失败。

    用法::

        class GetDeviceStatusTool(SpringToolMixin, Tool):
            tool_name = "get_device_status"
    """

    tool_name: str = "unknown"

    @property
    def log(self) -> ToolLog:
        if not hasattr(self, "_tool_log"):
            self._tool_log = ToolLog(self.tool_name)
        return self._tool_log

    def spring_url(self) -> str:
        return spring_url_from_credentials(self.runtime.credentials)

    def headers(self, *, json_content: bool = True) -> dict[str, str]:
        return request_headers_from_credentials(
            self.runtime.credentials, json_content=json_content
        )

    def base_ctx(self, **extra: Any) -> dict[str, Any]:
        ctx: dict[str, Any] = {"spring_url": self.spring_url()}
        ctx.update(extra)
        return ctx

    def require_spring_url(self, ctx: dict[str, Any]) -> str | None:
        if self.spring_url():
            return None
        self.log.info("参数校验失败 spring_service_url 为空")
        return self.log.error_text(
            "spring_service_url 为空", ctx, "插件凭证未配置服务地址"
        )

    def require_param(
        self,
        ctx: dict[str, Any],
        name: str,
        value: Any,
        detail: str,
        hint: str = "",
    ) -> str | None:
        if value:
            return None
        self.log.info(f"参数校验失败 {name} 为空")
        return self.log.error_text(f"{name} 为空", ctx, detail, hint=hint)

    def run_http(
        self,
        ctx: dict[str, Any],
        method: str,
        url: str,
        *,
        timeout: int = 15,
        params: dict[str, Any] | None = None,
        json_body: Any = None,
        headers: dict[str, str] | None = None,
        http_error_hint: str = "",
    ) -> tuple[requests.Response | None, str | None]:
        """执行 HTTP 请求。成功返回 (response, None)，失败返回 (None, error_text)。"""
        req_headers = headers or self.headers()
        self.log.log_request(method, url, params=params, payload=json_body)
        try:
            response = requests.request(
                method,
                url,
                headers=req_headers,
                params=params,
                json=json_body,
                timeout=timeout,
            )
            self.log.log_response(response)
            response.raise_for_status()
            return response, None
        except requests.exceptions.HTTPError as e:
            status = e.response.status_code if e.response is not None else "unknown"
            self.log.info(f"HTTP错误 status={status}")
            self.log.exception("HTTP错误")
            return None, self.log.error_text(
                f"HTTP 请求失败 status={status}",
                ctx,
                self.log.http_error_detail(e),
                hint=http_error_hint,
            )
        except requests.exceptions.ConnectionError as e:
            self.log.info(f"连接失败 url={url} error={e}")
            self.log.exception("连接失败")
            return None, self.log.error_text(
                "无法连接业务服务",
                ctx,
                str(e),
                hint="请确认 spring_service_url 为局域网 IP 且服务已启动",
            )
        except requests.exceptions.Timeout:
            self.log.info(f"请求超时 url={url} timeout={timeout}s")
            self.log.exception("请求超时")
            return None, self.log.error_text(
                "请求超时",
                ctx,
                f"超过 {timeout} 秒未响应",
            )
        except Exception as e:
            self.log.info(f"请求失败 error={e}")
            self.log.exception("请求失败")
            return None, self.log.error_text("工具执行异常", ctx, str(e))

    def fetch_log_credentials(self) -> bool:
        try:
            self.log.info(f"credential keys = {list(self.runtime.credentials.keys())}")
            self.log.info(f"spring_service_url = {self.spring_url()!r}")
            self.log.info(
                f"credentials = {safe_credentials_for_log(dict(self.runtime.credentials))}"
            )
            return True
        except Exception as e:
            self.log.info(f"读取凭证异常 error={e}")
            self.log.exception("读取凭证异常")
            return False

    def fetch_get_json(
        self,
        url: str,
        *,
        params: dict[str, Any] | None = None,
        timeout: int = 15,
    ) -> Any | None:
        """配置期 GET 请求，失败记日志并返回 None。"""
        self.log.log_request("GET", url, params=params)
        try:
            response = requests.get(
                url,
                headers=self.headers(json_content=False),
                params=params,
                timeout=timeout,
            )
            self.log.log_response(response)
            response.raise_for_status()
            return response.json()
        except requests.exceptions.ConnectionError as e:
            self.log.info(f"连接失败 url={url} error={e}")
            self.log.exception("连接失败")
        except requests.exceptions.Timeout:
            self.log.info(f"请求超时 url={url} timeout={timeout}s")
            self.log.exception("请求超时")
        except Exception as e:
            self.log.info(f"请求失败 url={url} error={e}")
            self.log.exception("fetch 请求失败")
        return None


# Provider 等非 Tool 场景可直接用 ToolLog
def create_tool_logger(tool_name: str) -> tuple[logging.Logger, Any]:
    tl = ToolLog(tool_name)
    return tl.logger, tl.info


def log_http_response(_log: Any, response: requests.Response, label: str = "响应") -> None:
    status = response.status_code
    body_text = response.text or ""
    _log(f"{label} 状态码 = {status}")
    if status >= 400:
        _log(f"{label} 错误响应体 = {truncate_for_log(body_text)}")
    else:
        _log(f"{label} 响应体 = {truncate_for_log(body_text)}")


def error_message(
    title: str,
    params: dict[str, Any],
    detail: str,
    hint: str = "",
) -> str:
    return ToolLog("compat").error_text(title, params, detail, hint=hint)
```

## 七、API 参考

### 7.1 工具函数

| 函数 | 说明 |
|------|------|
| `truncate_for_log(text, max_len=200)` | 截断日志文本 |
| `safe_params_for_log(params)` | 入参脱敏，长字符串截断，敏感字段打 `***` |
| `safe_credentials_for_log(credentials)` | 凭证脱敏 |
| `spring_url_from_credentials(credentials)` | 读取 `spring_service_url` |
| `request_headers_from_credentials(credentials)` | 构建 JSON 请求头，自动加 Bearer Token |

### 7.2 ToolLog 类

| 方法 | 说明 |
|------|------|
| `info(msg)` | 写 INFO 日志，带 `[tool_name]` 前缀 |
| `exception(msg)` | 写异常堆栈 |
| `invoke_enter(tool_parameters)` | 运行期入口日志 |
| `invoke_exit_success()` | 运行期成功出口日志 |
| `fetch_enter(parameter)` | 配置期入口日志 |
| `fetch_exit()` | 配置期出口日志 |
| `log_request(method, url, params, payload)` | 打印请求摘要 |
| `log_response(response)` | 打印状态码 + 响应体前 200 字符 |
| `error_text(title, params, detail, hint)` | 生成给前端的错误文本 |
| `http_error_detail(exc)` | 从 HTTPError 提取错误详情 |

### 7.3 SpringToolMixin 类

| 方法/属性 | 说明 |
|-----------|------|
| `tool_name` | 类属性，与 YAML `identity.name` 一致 |
| `self.log` | 懒加载 `ToolLog` 实例 |
| `spring_url()` | 当前运行时凭证中的服务地址 |
| `headers()` | 当前运行时请求头 |
| `base_ctx(**extra)` | 构建错误上下文 dict |
| `require_param(ctx, name, value, detail, hint)` | 必填校验，失败返回 `error_text` |
| `require_spring_url(ctx)` | 凭证地址校验 |
| `run_http(ctx, method, url, ...)` | 运行期 HTTP，返回 `(response, err)` |
| `fetch_log_credentials()` | 配置期打印脱敏凭证，失败返回 `False` |
| `fetch_get_json(url, params)` | 配置期 GET，失败返回 `None` |

## 八、使用示例

### 8.1 运行期工具 最简 GET 模板

```python
from typing import Any, Generator

from dify_plugin import Tool
from dify_plugin.entities.tool import ToolInvokeMessage

from .tool_logging import SpringToolMixin


class GetDeviceStatusTool(SpringToolMixin, Tool):
    tool_name = "get_device_status"

    def _invoke(
        self, tool_parameters: dict[str, Any]
    ) -> Generator[ToolInvokeMessage, None, None]:
        self.log.invoke_enter(tool_parameters)

        device_id = (tool_parameters.get("device_id") or "").strip()
        ctx = self.base_ctx(device_id=device_id)

        # 参数校验：失败则 yield 错误文本给前端
        if msg := self.require_param(
            ctx, "device_id", device_id, "未提供设备 ID",
            hint="请在工具节点中填写 device_id",
        ):
            yield self.create_text_message(msg)
            return
        if msg := self.require_spring_url(ctx):
            yield self.create_text_message(msg)
            return

        # HTTP 调用：自动打日志 + 异常处理
        url = f"{self.spring_url()}/api/devices/{device_id}/status"
        response, err = self.run_http(
            ctx, "GET", url, http_error_hint="请确认设备 ID 是否存在",
        )
        if err:
            yield self.create_text_message(err)
            return

        # 业务逻辑
        status_data = response.json()
        text = f"设备 {device_id} 状态: {status_data.get('status', '-')}"

        self.log.invoke_exit_success()
        yield self.create_text_message(text)
        yield self.create_json_message(status_data)
```

### 8.3 配置期 dynamic-select 模板

```python
from dify_plugin import Tool
from dify_plugin.entities import I18nObject, ParameterOption

from .tool_logging import SpringToolMixin


class DynamicDeviceQueryTool(SpringToolMixin, Tool):
    tool_name = "dynamic_device_query"

    def _fetch_parameter_options(self, parameter: str) -> list[ParameterOption]:
        self.log.fetch_enter(parameter)

        if not self.fetch_log_credentials():
            return []

        if parameter != "device_id":
            self.log.info(f"跳过非目标参数 parameter={parameter!r}")
            return []

        if not self.spring_url():
            self.log.info("spring_service_url 为空")
            return []

        url = f"{self.spring_url()}/api/dify-plugin/device-select-options"
        items = self.fetch_get_json(url)
        if items is None:
            return []

        options = [
            ParameterOption(
                value=str(item["value"]),
                label=I18nObject(
                    en_US=str(item.get("label", item["value"])),
                    zh_Hans=str(item.get("label", item["value"])),
                ),
            )
            for item in items
            if item.get("value")
        ]
        self.log.info(f"返回 options count={len(options)}")
        self.log.fetch_exit()
        return options
```

配置期 **不要** `create_text_message`，失败时返回 `[]`，用户看到下拉为空，通过 daemon 日志排查。

### 8.4 Provider 凭证校验

Provider 不是 Tool，直接用 `ToolLog`：

```python
from tools.tool_logging import ToolLog, safe_credentials_for_log, spring_url_from_credentials

_log = ToolLog("iot_device_plugin")


class IotDevicePluginProvider(ToolProvider):
    def _validate_credentials(self, credentials: dict[str, Any]) -> None:
        _log.info("===== _validate_credentials ENTER =====")
        _log.info(f"credentials = {safe_credentials_for_log(credentials)}")

        spring_url = spring_url_from_credentials(credentials)
        if not spring_url:
            raise ToolProviderCredentialValidationError("Spring服务地址不能为空")

        _log.info(f"凭证格式校验通过 spring_url={spring_url!r}")
        _log.info("===== _validate_credentials EXIT =====")
```

### 8.5 特殊场景：HTTP 4xx 也要返回给前端（generic_http）

若业务需要把非 2xx 响应展示给用户而非抛异常，**不要**用 `run_http`（内部会 `raise_for_status`），改用：

```python
self.log.log_request(method, url, params=params, payload=body)
response = requests.request(method, url, ...)
self.log.log_response(response)
# 自行格式化 status_code 和 response.text
```

## 十一、常见问题

### Q1 用了 print 为什么 kubectl 看不到

服务器安装模式下 stdout/stderr 不转发到容器日志。必须用 `ToolLog` / `plugin_logger_handler`。

### Q2 Spring 有日志但插件日志没有

说明旧版用了 `print(stderr)`。升级到含 `tool_logging.py` 的版本即可。

### Q3 Multiple subclasses of Tool

`SpringToolMixin` 或其父类继承了 `Tool`。确保 Mixin **不继承** Tool，工具类写 `(SpringToolMixin, Tool)`。

### Q4 运行工作流为什么看不到 fetch 日志

`_fetch_parameter_options` 只在编辑器拉下拉时触发，运行期走 `_invoke`。

### Q5 如何适配非 Spring 后端

修改 `spring_url_from_credentials` 和 `request_headers_from_credentials` 中的凭证字段名，或在本插件内新增自己的 Mixin。
