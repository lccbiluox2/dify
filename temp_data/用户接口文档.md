# Dify 用户相关接口文档

本文档整理了 Dify 项目中所有与用户（Account）相关的 API 接口。

**Base URL**: `http://{host}:{port}/console/api`

**认证方式**: 大多数接口需要携带 `access_token`（Cookie）和 `x-csrf-token`（Header）

## 1. 用户认证相关

### 1.1 用户登录

**接口**: `POST /console/api/login`

**描述**: 用户账号密码登录

**请求体**:
```json
{
  "email": "user@example.com",
  "password": "string",
  "remember_me": false,
  "invite_token": "string (optional)"
}
```

**响应**: 
- 成功后在 Cookie 中设置 `access_token`, `refresh_token`, `csrf_token`
- 返回 `{"result": "success"}`

**错误码**:
- `account_in_freeze`: 账号冻结
- `email_password_login_limit`: 登录频率限制
- `invalid_email`: 邮箱无效
- `authentication_failed`: 认证失败
- `account_banned`: 账号被封禁

---

### 1.2 用户登出

**接口**: `POST /console/api/logout`

**描述**: 用户登出，清除认证信息

**请求体**: 无

**响应**: 
```json
{"result": "success"}
```

**说明**: 会清除 Cookie 中的 `access_token`, `refresh_token`, `csrf_token`

---

### 1.3 刷新令牌

**接口**: `POST /console/api/refresh-token`

**描述**: 使用 refresh_token 刷新访问令牌

**请求**: 从 Cookie 中读取 `refresh_token`

**响应**: 
```json
{"result": "success"}
```

**说明**: 会在 Cookie 中设置新的 `access_token`, `refresh_token`, `csrf_token`

---

## 2. 用户信息管理

### 2.1 获取用户信息

**接口**: `GET /console/api/account/profile`

**描述**: 获取当前登录用户的详细信息

**请求头**:
```
Accept: */*
content-type: application/json
x-csrf-token: {token}
Cookie: access_token={token}
```

**响应**:
```json
{
  "id": "b0c8e84c-39c9-4e8c-bec7-a77b54e182ac",
  "name": "ailpha",
  "email": "admin@example.com",
  "is_password_set": true,
  "interface_language": "zh-Hans",
  "interface_theme": "light",
  "timezone": "Asia/Shanghai",
  "last_login_at": 1778461403,
  "last_login_ip": "10.23.17.18",
  "created_at": 1777080165,
  "avatar": null,
  "avatar_url": null
}
```

**字段说明**:
- `id`: 用户唯一标识
- `name`: 用户名
- `email`: 邮箱
- `is_password_set`: 是否已设置密码
- `interface_language`: 界面语言（`zh-Hans`, `en-US` 等）
- `interface_theme`: 界面主题（`light`, `dark`）
- `timezone`: 时区
- `last_login_at`: 最后登录时间戳
- `last_login_ip`: 最后登录 IP
- `created_at`: 创建时间戳
- `avatar`: 头像文件 ID
- `avatar_url`: 头像 URL

## 3. 密码管理

### 3.1 修改密码

**接口**: `POST /console/api/account/password`

**描述**: 修改用户密码

**请求体**:
```json
{
  "password": "旧密码",
  "new_password": "新密码",
  "repeat_new_password": "重复新密码"

```

**响应**: 返回更新后的用户信息对象

- `current_password_incorrect`: 当前密码错误
- `repeat_password_not_match`: 两次输入的新密码不一致

## 4. 邮箱管理

### 4.1 修改邮箱 - 发送验证邮件

**接口**: `POST /console/api/account/change-email`

**描述**: 发送邮箱修改验证邮件

**请求体**:
```json
{
  "email": "newemail@example.com",
  "language": "zh-Hans (optional)",
  "phase": "old_email (optional)",
  "token": "string (optional, new_email 阶段需要)"
}
```

**响应**:
```json
{
  "result": "success",
  "data": "token_string"
}
```

**说明**: 
- 需要经历两个阶段：`old_email`（验证旧邮箱）→ `new_email`（验证新邮箱）
- 第二阶段需要传入第一阶段返回的 token

---

### 4.2 修改邮箱 - 验证验证码

**接口**: `POST /console/api/account/change-email/validity`

**描述**: 验证邮箱修改验证码

**请求体**:
```json
{
  "email": "user@example.com",
  "code": "123456",
  "token": "string"
}
```

**响应**:
```json
{
  "is_valid": true,
  "email": "user@example.com",
  "token": "new_token_string"
}
```

---

### 4.3 修改邮箱 - 完成修改

**接口**: `POST /console/api/account/change-email/reset`

**描述**: 完成邮箱修改

**请求体**:
```json
{
  "new_email": "newemail@example.com",
  "token": "string"
}
```

**响应**: 返回更新后的用户信息对象

**错误码**:
- `account_in_freeze`: 新邮箱在冻结期
- `email_already_in_use`: 新邮箱已被使用
- `invalid_token`: 令牌无效

---

### 4.4 检查邮箱唯一性

## 7. 第三方集成

### 7.1 获取第三方集成信息

**接口**: `GET /console/api/account/integrates`

**描述**: 获取用户绑定的第三方账号信息

**响应**:
```json
{
  "data": [
    {
      "id": "integrate_id",
      "provider": "github",
      "created_at": 1234567890,
      "is_bound": true,
      "link": null
    },
    {
      "id": null,
      "provider": "google",
      "created_at": null,
      "is_bound": false,
      "link": "http://host/console/api/oauth/login/google"
    }
  ]
}
```

**字段说明**:
- `provider`: 第三方提供商（`github`, `google`）
- `is_bound`: 是否已绑定
- `link`: 未绑定时，绑定链接地址

## 附录

### A. 通用错误码

| 错误码 | 说明 |
|--------|------|
| `account_in_freeze` | 账号在冻结期 |
| `account_not_found` | 账号不存在 |
| `account_banned` | 账号被封禁 |
| `email_already_in_use` | 邮箱已被使用 |
| `invalid_email` | 邮箱格式无效 |
| `invalid_token` | 令牌无效 |
| `email_code_error` | 验证码错误 |
| `current_password_incorrect` | 当前密码错误 |
| `repeat_password_not_match` | 两次密码不一致 |
| `email_send_ip_limit` | IP 发送邮件频率限制 |
| `login_rate_limit` | 登录频率限制 |
| `invalid_invitation_code` | 邀请码无效 |

### B. 认证要求

大多数接口需要：
1. 已登录状态（Cookie 中包含 `access_token`）
2. 正确的 CSRF Token（Header 中 `x-csrf-token`）
3. 部分接口需要 `account_initialization_required`（已完成初始化）

### C. 支持的语言

- `zh-Hans`: 简体中文
- `en-US`: 英语（美国）
- 其他语言请参考 `constants/languages.py`

### D. 支持的时区

标准 IANA 时区格式，例如：
- `Asia/Shanghai`
- `America/New_York`
- `Europe/London`
