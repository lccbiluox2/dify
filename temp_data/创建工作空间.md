# 创建工作空间分析

## 工作空间(Tenant)创建方式

在 Dify 系统中,工作空间即 `Tenant`。有以下几种创建方式:

### 1. **注册时自动创建个人工作空间**
当用户注册时,如果允许创建工作空间,会自动创建个人工作空间:

```python
# 在 AccountService.create_account_and_tenant() 中
TenantService.create_owner_tenant_if_not_exist(account=account)
```

核心逻辑在 `TenantService.create_owner_tenant_if_not_exist`:
- 检查用户是否已有工作空间
- 检查系统是否允许创建工作空间 (`is_allow_create_workspace`)
- 检查工作空间配额是否可用
- 创建租户(默认名称: `"{用户名}'s Workspace"`)
- 将用户添加为 owner 角色
- 创建默认积分池

### 2. **管理员后台创建**
通过 `/console/api/all-workspaces` 接口(需要管理员权限)可以查看所有工作空间,但代码中**没有直接提供创建工作空间的公开 API**。

### 3. **内部创建方法**
`TenantService.create_tenant` 是底层创建方法:

```python
@staticmethod
def create_tenant(name: str, is_setup: bool | None = False, is_from_dashboard: bool | None = False) -> Tenant:
    """Create tenant"""
    # 检查是否允许创建
    if not FeatureService.get_system_features().is_allow_create_workspace and not is_setup and not is_from_dashboard:
        raise NotAllowedCreateWorkspace()
    
    tenant = Tenant(name=name)
    db.session.add(tenant)
    db.session.commit()
    
    # 创建插件升级策略
    # 生成加密密钥
    # 创建默认积分池
    
    return tenant
```

### 4. **邀请成员时创建**
通过 `RegisterService.invite_new_member` 邀请新用户时:
- 如果用户不存在,会先注册账号
- 然后将其添加为租户成员

### 关键点

⚠️ **当前代码中没有提供公开的"创建工作空间" API 接口**

现有的工作空间相关 API:
- `GET /console/api/workspaces` - 获取用户加入的所有工作空间列表
- `POST /console/api/workspaces/current` - 获取当前工作空间信息  
- `POST /console/api/workspaces/switch` - 切换工作空间
- `POST /console/api/workspaces/info` - 修改工作空间名称
- `POST /console/api/workspaces/custom-config` - 自定义配置

## 如何创建工作空间

如果需要创建工作空间的功能,需要:

### 步骤1: 添加新的 API 路由
在 `api/controllers/console/workspace/workspace.py` 中添加新的路由类:

```python
@console_ns.route("/workspaces/create")
class CreateWorkspaceApi(Resource):
    @console_ns.expect(console_ns.models[CreateWorkspacePayload.__name__])
    @setup_required
    @login_required
    @account_initialization_required
    def post(self):
        current_user, _ = current_account_with_tenant()
        payload = console_ns.payload or {}
        args = CreateWorkspacePayload.model_validate(payload)
        
        # 检查是否允许创建工作空间
        if not FeatureService.get_system_features().is_allow_create_workspace:
            raise NotAllowedCreateWorkspace()
        
        # 创建工作空间
        tenant = TenantService.create_tenant(name=args.name, is_from_dashboard=True)
        
        # 将当前用户添加为 owner
        TenantService.create_tenant_member(tenant, current_user, role="owner")
        
        return {
            "result": "success",
            "workspace": marshal(WorkspaceService.get_tenant_info(tenant), tenant_fields)
        }
```

### 步骤2: 定义请求数据模型
```python
class CreateWorkspacePayload(BaseModel):
    name: str = Field(..., min_length=1, max_length=255, description="工作空间名称")
```

### 步骤3: 注册 Schema 模型
在 `register_schema_models` 中添加 `CreateWorkspacePayload`。

### 步骤4: 权限控制
- 使用 `@cloud_edition_billing_resource_check` 检查工作空间配额(如果是云版本)
- 检查用户是否有权创建新工作空间
- 可添加创建数量限制

### 核心服务方法

#### TenantService.create_tenant
位置: `api/services/account_service.py`

主要功能:
1. 创建 Tenant 记录
2. 创建插件自动升级策略
3. 生成加密公钥
4. 创建默认积分池

#### TenantService.create_tenant_member
位置: `api/services/account_service.py`

主要功能:
- 将用户添加为租户成员
- 支持指定角色(owner/admin/normal等)
- 一个租户只能有一个 owner

### 注意事项

1. **权限检查**: 需要验证 `FeatureService.get_system_features().is_allow_create_workspace`
2. **配额限制**: 企业版需要检查工作空间数量限制
3. **角色分配**: 创建者通常自动成为 owner
4. **事件触发**: 创建成功后会触发 `tenant_was_created` 事件
5. **默认配置**: 自动创建插件升级策略和积分池
